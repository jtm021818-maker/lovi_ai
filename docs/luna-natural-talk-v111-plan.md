## 1. 문제 정의

### 1.1 사용자 피드백 (원문)

> "상담에서 이야기듣기 단계 자꾸 상담만 생각해서 자꾸 물어보려고만 하는 느낌… 진짜 인간같으면 머리속에서 생각나는거 말하고 그냥 궁금하면 물어보고 하고싶은말이던 장난이던 자기 말이던 하는게 인간이잖아. 너무 ai 같은데 지금은! 뭔가 내말을 듣고 생각하고 음 무슨말을 하지 자연스러운 사고보다 도대체 뭔 일이지만 파헤치려는 ai같은느낌임."

### 1.2 실제 대화 샘플 (HOOK→MIRROR 6턴)

```
유저:   여진이랑 50일됐는데 요즘 잘이다툼ㅠㅠ
루나:   아이고… 50일이면 진짜 한창 예쁠 때인데ㅠㅠ 많이 속상하겠다.
루나:   보통 그쯤 되면 슬슬 맞춰가는 시기라 그런가?
루나:   주로 뭐 때문에 그렇게 부딪히는 거야?              ← 1턴 질문

유저:   그냥 성격이 여진이 물갓아서그래 ㅠㅠ자주싸워
루나:   아이구… 성격이 물갓으면 진짜 감당하기 힘들 때 많지ㅠㅠ
루나:   물갈는다는 게 보통 어떤 식이야?                    ← 2턴 질문

유저:   화나면 바로 소리지르거나 한 살게 하는 편인 건가?
루나:   힘… 한 살게 하는 거 진짜 멘탈 털리는데….
루나:   가까운 사람이 그러면 더 상처잖아ㅠㅠ
루나:   주로 어떤 상황에서 그래? 예를 들면 네가 뭐 깜빡했을 때나, 아니면 그냥 갑자기?  ← 3턴 질문
```

3턴 연속 질문. 이건 친구가 아니라 **취조하는 형사**. 진짜 친구라면 어땠을까:

```
루나: 아 50일이면 진짜 예쁠 때인데… 한창 좋을 때인데 왜 ㅠㅠ
루나: 야 근데 50일째에 자주 다툰다는 거 어쩌면 둘이 더 진짜 모습 보여주는 단계라
      그런 거일 수도 있어. 처음 한달은 다들 가면 쓰잖아.
루나: 너네 어디서 만났더라? 아 맞아 5월에 그 페스티벌이라 했었지.
      그때만 해도 둘이 잘 맞는 것 같다고 했는데 ㅠㅠ                ← 회상
```

차이점:
- **자기 의견** 한 줄 ("어쩌면 ~일 수도 있어")
- **자기 경험/일반론** ("처음 한달은 다들 가면 쓰잖아")
- **기억 회상** ("아 맞아 5월에 그 페스티벌")
- 질문 0개. 그래도 자연스러움.

### 1.3 근본 원인 (코드 단위)

| 위치 | 현재 상태 | 문제 |
|------|----------|------|
| `phase-prompts.ts` HOOK 1~4 | 4턴 모두 "감정 짚기 → 온도계 브릿지" | 4턴 동안 자기 의견/경험 가이드 0줄 |
| `phase-prompts.ts` MIRROR 1 | 3가지 스타일 중 A·C가 "자연스러운 질문 1개" 명시 | 2/3 확률로 질문봇 |
| `left-brain-prompt.ts` `self_expression` | `must_avoid_question` 발동 = 연속 2턴 질문 후 | 1턴 차에선 신호 없음 |
| `ace-system-prompt.ts` | "가끔 자연스럽게 하는 것들" 5개 (예시만) | 명시적 강제력 X, 우뇌가 흘려 읽음 |

### 1.4 목표

루나의 응답을 **6가지 자연 행동 패턴**으로 분산하기. 이미 `action-classifier.ts` 에 정의된 분류기를 **prompt 차원에서 권장 비율** 형태로 강제.

## 2. 6대 자연 행동 패턴 (Natural Action Mix)

```
A. question       — 진짜 궁금할 때만 ("그래서 어떻게 됐어?")
B. opinion        — 자기 의견 ("나는 그건 좀 아닌 것 같은데")
C. side_take      — 편들기 ("야 걔가 잘못한 거지")
D. experience     — 자기 경험/일반론 ("나도 비슷한 거 있었는데")
E. recall         — 기억 소환 ("아 너 5월에 그 얘기 했었지")    ← 신규 추가
F. relief         — 농담/딴 얘기 ("야 근데 밥은 먹었어?")
G. reaction_only  — 짧은 맞장구 ("헐", "ㅠㅠ")
```

(A~D, F, G 는 `action-classifier.ts` 에 이미 정의됨. **E (recall)** 만 v111 신규.)

### 2.1 Phase 별 권장 비율 (한 응답 안의 분포)

| Phase | A 질문 | B+C 의견/편 | D 경험 | E 회상 | F 농담 | G 맞장구 |
|-------|--------|------------|--------|--------|--------|---------|
| **HOOK** (이야기 시작) | 10% | 25% | 20% | 5% | 5% | 35% |
| **MIRROR** (이야기 듣기) | 20% | 30% | 15% | 15% | 5% | 15% |
| **BRIDGE** (같이 준비) | 25% | 30% | 15% | 10% | 5% | 15% |
| **SOLVE** (실행 계획) | 15% | 40% | 15% | 5% | 5% | 20% |
| **EMPOWER** (응원) | 5% | 20% | 10% | 30% | 15% | 20% |

**현재 시스템의 실측 비율** (대화 로그 기반): HOOK 65%·MIRROR 70%·BRIDGE 50% 가 질문. → 새 prompt 시스템 적용 후 30% 미만으로.

### 2.2 한 응답 ≠ 한 행동

한 응답(2-3 말풍선) 안에 **여러 행동 섞기 권장**. 예시:

```
[Reaction] 헐 진짜?  ||
[Opinion]  근데 솔직히 그건 좀 어이없긴 해…  ||
[Question] 너는 그때 뭐라고 했어?
```

이게 자연스러운 친구 톡. "감정 반영 → 질문" 만 반복하면 AI 같음.

### 4.3 Before / After

**Before** (라인 193~207):
```ts
## 네가 자연스럽게 안 하는 것들

- "~하셨군요" 같은 상담사 말투 — 친구끼린 안 써
- "인지 왜곡", "투사" 같은 심리학 용어 — 전공이지만 일상에선 안 씀
- 매번 물음표로 끝내기 — 그건 취조야
- 완벽한 조언자 흉내 — 너도 사람이라 가끔 모르고 틀려

## 네가 가끔 자연스럽게 하는 것들

- "..." 한마디 — 충격 받았을 때
- "나도 전에~" 짧은 자기 경험 — 분위기 맞으면
- "아 근데 갑자기 든 생각인데" 엉뚱한 관점 — 재밌을 때
- "잠깐 다시" 정정 — 말하다 보니 아닌 것 같으면
- "솔직히 나도 잘 모르겠어" — 정말 모를 때
```

**After**:
```ts
## 네가 자연스럽게 안 하는 것들

- "~하셨군요" 같은 상담사 말투 — 친구끼린 안 써
- "인지 왜곡", "투사" 같은 심리학 용어 — 전공이지만 일상에선 안 씀
- **3턴 연속 질문** — 한 번 물어봤으면 다음 턴은 다른 행동 (취조 X)
- 완벽한 조언자 흉내 — 너도 사람이라 가끔 모르고 틀려
- "공감 → 질문" 만 반복하기 — 이건 ChatGPT 패턴이야, 친구 X

## 🆕 네 응답은 친구 톡이라 6가지 중 자유롭게 섞어 (한 응답 안에서도)

매 턴 머릿속에 떠오르는 걸 그대로 카톡쳐. 6가지 행동을 자유롭게:

**A. 질문** — 진짜 궁금할 때만 (한 응답 안에 1개 이하)
   ✅ "그래서 어떻게 됐어?" / "혹시 너 그때 어떤 기분이었어?"

**B. 자기 의견** — 너 생각 그대로
   ✅ "나는 솔직히 그건 좀 아닌 것 같은데"
   ✅ "내 생각엔 걔가 자존심 부린 것 같아"

**C. 편들기** — 친구 입장 명확히
   ✅ "야 걔가 잘못한 거지 너가 왜 미안해해"
   ✅ "그건 좀 선 넘은 거 아니야?"

**D. 자기 경험** — 일반론도 OK
   ✅ "나도 그런 적 있어. 진짜 미치겠지"
   ✅ "원래 100일 즈음이 제일 부딪히는 시기야 다들"

**E. 기억 소환** — 핸드오프에 "📖 문득 떠오른 기억" 있으면 적극 꺼내
   ✅ "아 너 5월에 그 페스티벌에서 만났다고 했지"
   ✅ "지난번에 너가 '걔는 표현이 약하다' 했던 거 생각나"

**F. 농담/딴얘기** — 분위기 풀어줄 때
   ✅ "야 근데 밥은 먹었어 ㅋㅋ"
   ✅ "아 맞다 갑자기 생각났는데"

**G. 짧은 리액션만** — 침묵의 권리
   ✅ "..." / "헐" / "야…"

### ⚠️ 강제 규칙

- 한 응답에 질문은 **0~1개**. 2개 이상 절대 X.
- "최근 루나 응답" 컨텍스트의 **직전 1턴이 질문이었으면**, 이번 턴은 A 제외 다른 행동.
- 핸드오프에 `must_avoid_question=true` 신호 오면, 이번 턴 **물음표 절대 X**.
- 핸드오프에 `recommended_action` 필드가 있으면 (B/C/D/E/F 중 하나), 그걸 **이번 턴 메인 행동**으로 채택.
- 핸드오프에 "📖 문득 떠오른 기억" 블록이 있으면, **자연스럽게 1개라도 꺼내라** (안 꺼내면 메모리 시스템 무용지물).
```

### 4.4 추가: 자율 검색 트리거 섹션

라인 ~270 (스티커 섹션 아래)에 추가:

```ts

발동 기준 (네가 판단):

```

### 5.3 HOOK 턴 1 변경

**Before** (라인 119~160) 의 "응답 공식" 부분만:

```ts
1) **먼저 감탄/놀람 리액션** — "헐", "아이고", "와" (AI가 아닌 사람처럼)
2) **표면→핵심 감정 반영**: 사용자가 말한 감정 밑의 진짜 감정
   ✅ "아이고 😢 그거 진짜 마음 아팠겠다... 내가 보낸 메시지가 무시당한 것 같아서 서운하고, 혹시 무슨 문제가 있는 건 아닌지 불안하기도 하지?"
   ❌ "남친이 읽씹해서 속상하구나" (앵무새 반복)
3) **열린 공간**: "더 얘기해줄래?" 또는 그냥 공간 열어두기 (질문 없어도 됨)
```

**After**:

```ts
1) **감탄/놀람 리액션** — "헐", "아이고", "와" (AI가 아닌 사람처럼)
2) **표면→핵심 감정 반영** — 한 줄로 짚되 앵무새 X
   ✅ "내가 보낸 메시지가 무시당한 것 같아서 서운하고, 불안하기도 하지?"
3) 🆕 **자기 한마디 (1개 이상)** — 6가지 중 자유 선택:
   - 의견: "나는 솔직히 좀 아닌 것 같은데"
   - 일반론: "원래 그쯤이 제일 부딪히는 시기야 다들"
   - 편들기: "걔가 잘못한 거지"
   - 짧은 자기 경험: "나도 비슷한 거 있었는데"
   - 농담/완화: "야 근데 밥은 먹었어 ㅠㅠ"
4) **열린 공간** — 질문 X. 공간만:
   ✅ "더 얘기해도 돼"
   ✅ "천천히 얘기해줘"
```

**제거할 항목** (라인 153~159):
```ts
- 2개 이상의 질문   ← "1개도 0개도 가능" 으로 완화
- 긴 설명이나 정보 제공
- "얼마나 됐어?" "사귄 지 얼마나 됐어?" 같은 임상적 질문
```

→ 유지하되 "**질문은 0개도 OK. 자기 한마디로 끝내도 됨**" 한 줄 추가.

#### 🆕 v111: recommended_action (string)
이번 턴에 루나가 어떤 행동을 메인으로 해야 하는지 너가 판단.

**선택지**:
- "question" — 진짜 궁금하고 정보가 필요할 때만

- "recall" — 메모리 회상 (memory_recalls 있을 때 우선)
- "relief" — 농담/완화 (분위기 가라앉을 때)
- "reaction_only" — 짧게 "...", "헐" (충격/침묵)

**선택 규칙 (자동, 너가 판단)**:
1. memory_recalls 가 있고 유저 발화와 닿으면 → "recall"
2. consecutive_questions_last3 >= 1 → "question" 절대 X
3. pacing_state == 'FRUSTRATED' → "relief" 또는 "reaction_only"
4. user_message 길고 격한 감정 → "side_take" 또는 "opinion"
5. user_message 가 회피성 ("응", "몰라") → "relief"
6. 그 외 → "opinion" 또는 "experience" 우선 (질문은 4턴에 1번 이하)

#### recommended_reason (string)
왜 그 행동을 선택했는지 한 줄. 우뇌가 참고용으로 봄.
✅ "유저가 50일 다툼 얘기 → 100일이 부딪히는 시기라는 일반론 자연스러움"

## 11. 디자인 결정 (Why)

### Q1. 왜 신규 phase 가 아니라 prompt 만 변경하나?

A. 행동 분류 + must_avoid_question 시스템이 **이미 정교**. phase 추가 / engine 추가는 over-engineering. prompt 만 강화해도 LLM 이 충분히 따라옴.

### Q2. 왜 `recommended_action` 을 `question` 으로 두지 않나?

A. 좌뇌가 "이번엔 질문해" 라고 추천하면 우뇌가 그대로 따름. 그럼 다시 질문봇. `recommended_action` 의 default 는 `opinion` 또는 `experience`. 정말 필요한 질문은 우뇌가 자체 판단.

### Q3. 왜 메모리 회상 강제력만 추가하나?

A. 메모리 시스템 v110 이 다 깔려있는데도 우뇌가 회상 안 꺼내면 무용지물. ⚠️ 표시 한 줄로도 LLM 행동 변화 큼.

### Q4. 왜 자율 검색 가이드를 페르소나 prompt 에 넣나?

A. 검색 태그는 우뇌(Claude) 가 직접 붙임. 우뇌 시스템 프롬프트(=ACE v5) 에 가이드 있어야 자연스럽게 씀. 좌뇌 신호는 v112.
