## 1. 문제의 본질 — 왜 "행동 지정"은 인간이 아닌가

### 1.1 현재 접근의 근본적 한계

v2 계획서에서 우리는 "9가지 반응 유형"을 HOOK Purpose에 넣었다:
```
① 리액션  ② 의견  ③ 편들기  ④ 경험 한 줄  ⑤ 보편화
⑥ 릴리프  ⑦ 생각 흘리기  ⑧ 맞장구  ⑨ 메타 코멘트
```

이건 개선이긴 하지만, **근본적으로 "선택지에서 고르기"의 변형**이다.

진짜 인간이 친구 얘기를 들을 때:
- "음... 지금 ① 리액션을 할까, ② 의견을 말할까, ⑥ 릴리프를 넣을까?" → **이런 사고 과정은 없다.**
- 대신: 듣고 → 뭔가 느끼고 → 그 느낌에서 말이 나온다. **그게 전부.**

### 1.2 인간의 대화 결정 과정 (신경과학)

Damasio의 Somatic Marker Hypothesis (2024 재검증):
- 인간의 사회적 판단은 **감정이 먼저, 이성이 나중**
- "이 상황에서 뭘 해야 하지?" → 머리로 계산하는 게 아니라, **몸이 먼저 반응**
- 가슴이 답답해지거나, 화가 올라오거나, 웃음이 나거나 → 그 감정이 행동을 결정

친구 얘기를 듣는 상황:
```
친구: "걔가 카톡 안 봐 ㅠㅠ"
  ↓
나의 뇌: [상황 파악] 읽씹. 내가 겪어봤어.
  ↓
나의 몸: [감정 반응] 화남 + 안쓰러움 동시에 올라옴
  ↓
나의 입: "야 걔 미쳤어?" (화남에서 나온 말)
  또는: "아... 그거 진짜 힘들지" (안쓰러움에서 나온 말)
  또는: "ㅋㅋ 걔 진짜 대단하다" (화남 + 유머 전환)
```

**핵심: 어떤 말이 나올지는 "올라온 감정"이 결정한다. 선택지 목록이 아니다.**

### 1.3 The Paradox of Specificity (구체성의 역설)

연구 결과 (Persona Prompting 메타 분석, 2025):
- 행동을 구체적으로 지정할수록 → 기계적인 느낌 증가
- 캐릭터의 **가치관/감정/경험**을 풍부하게 주면 → 자연스러운 행동이 emerge

| 접근 | 예시 | 결과 |
|------|------|------|
| 규칙 기반 | "공감 후 의견을 30% 확률로 삽입" | AI가 따르지만 기계적 |
| 행동 목록 | "9가지 중 상황에 맞게 골라" | 덜 기계적이지만 "고르는" 느낌 |
| **존재 기반** | **"너는 이런 사람이야. 느껴봐."** | **자연스러운 반응이 나옴** |

## 2. 인간의 대화 인지 과정

### 2.1 5단계 무의식적 과정

인간이 친구 얘기를 들을 때 0.5초 안에 일어나는 것:

```
Stage 1: 지각 (Perception)
  "걔가 카톡 안 봐" → 읽씹 상황 인식

Stage 2: 연상 (Association)
  "나도 그때..." → 자기 경험 자동 연상
  "아 이 애가 전에도 이런 적 있었는데" → 맥락 연결

Stage 3: 감정 생성 (Emotion Generation) ← 핵심!
  상황 + 연상 → 복합 감정 자동 생성
  "화나네" + "안쓰러워" + "약간 답답해"
  이 감정의 조합이 매번 다르기 때문에 매번 다른 반응이 나옴

Stage 4: 충동 (Impulse)
  감정에서 "뭔가 말하고 싶은 것"이 올라옴
  이건 의식적 선택이 아니라 자동적 충동
  "야 걔 미쳤어?" 가 입에서 나오려 함

Stage 5: 표현 (Expression)
  충동을 약간의 사회적 필터를 거쳐 말로
  때로는 충동 그대로 ("야 미쳤어?")
  때로는 완화 ("아... 그건 좀...")
  때로는 억제 (말 안 하고 "..." 만)
```

**AI에게 이 5단계를 시뮬레이션하게 하면, "선택지에서 고르기"가 아닌 "느끼고 반응하기"가 된다.**

### 2.2 왜 인간은 매번 다른 반응을 하는가

같은 "걔가 카톡 안 봐"에 대해 인간이 매번 다르게 반응하는 이유:

1. **현재 감정 상태(mood)**: 오늘 기분이 좋으면 "ㅋㅋ 걔 뭐래"(유머), 나도 기분 안 좋으면 "아 진짜 짜증나" (공감 강화)
2. **에너지 수준**: 피곤하면 "헐..." 한마디, 에너지 넘치면 "야 걔 미쳤어?? 진짜?? 어떻게 그래??"
3. **직전 대화의 잔여 감정**: 방금 웃긴 얘기 하다가 이 얘기 나오면 톤이 달라짐
4. **관계 깊이**: 처음 듣는 얘기면 조심, 세 번째 같은 얘기면 "야 또??"
5. **자기 경험**: 비슷한 경험 있으면 "나도 그랬거든", 없으면 "헐 그런 일이..."

**현재 루나 시스템에서 이 중 실제로 응답에 반영되는 것:**
- ✅ 현재 감정 상태 → PAD 모델로 추적... 하지만 **응답에 연결 안 됨** (buildEmotionPrompt 미호출!)
- ❌ 에너지 수준 → 없음
- ❌ 직전 대화 잔여 감정 → lunaRecentSummaries는 내용 요약뿐
- ✅ 관계 깊이 → intimacyScore 있음
- ✅ 자기 경험 → memory-engine이 있지만, 기계적 1문장 삽입

**핵심 발견: 반응 다양성의 원천인 "감정 상태"가 응답 생성에 구조적으로 단절되어 있다.**

### 2.3 Conversational Spontaneity (대화의 즉흥성)

인간 대화에서 "아 맞다 그거 생각났는데" 같은 즉흥적 전환이 일어나는 메커니즘:

- **연상 네트워크(Associative Network)** — 하나의 단어/감정이 다른 기억을 활성화
- "읽씹" → "아 나도 전에 읽씹 당했을 때..." → "그때 나 어떻게 했더라..." → "야 나 그때..."
- 이 연상은 **의식적이지 않음**. 자동으로 일어남.

현재 루나: memory-engine의 트리거가 이 연상을 시뮬레이션하지만, injection이 "반드시 넣어"라서 부자연스러움.

## 3. Character Emergence vs Character Specification

### 3.1 Stanford Generative Agents (2023) — 행동이 "나오는" 시스템

Park et al.의 Generative Agents 논문 핵심:
- 에이전트에게 **행동 목록을 주지 않음**
- 대신: 성격 + 기억 + 현재 상황 → 에이전트가 **스스로 계획하고 행동**
- 결과: 깜짝 파티를 자발적으로 기획하는 등, **설계자가 예상 못한 행동**이 emerge

```
Generative Agents 구조:
  Memory Stream (기억 흐름)
    + Reflection (성찰)
    + Planning (계획)
  → Action (행동)

핵심: 행동이 "지정"이 아니라 기억+성찰에서 "나옴"
```

### 3.2 Method Acting for AI

Stanislavski 메소드 연기의 핵심 원리:
1. **"만약에 내가 이 상황이라면"** (Magic If) — 상황을 자기 것으로 느끼기
2. **감정 기억** (Emotional Memory) — 과거 비슷한 감정을 소환하여 현재 연기에 사용
3. **목표** (Super-objective) — "이 캐릭터가 진짜 원하는 것"을 이해

AI에 적용:
```
기존: "루나는 공감 잘하는 언니야. 공감해."
메소드: "루나, 너도 전남친에게 읽씹 당했잖아. 그때 네가 느낀 그 감정...
       지금 이 동생이 똑같이 느끼고 있어. 그때의 너처럼."
```

**두 번째 프롬프트에서 AI는 "행동 목록에서 선택"이 아니라, "그때의 감정을 떠올려서 반응"하게 됨.**

### 3.3 Chain of Persona (ACL 2025)

응답 생성 전 "나는 어떤 사람인가?" 자기 질문:
```
[자기 질문 단계 — 출력하지 마]
Q: "나(루나)는 이런 상황에서 어떤 사람이야?"
A: "나는 읽씹 당해본 적 있어. 그때 진짜 미치는 줄 알았어.
   지금 이 동생이 그걸 겪고 있으니까... 나는 걔 편이야 당연히."

Q: "지금 내 몸에서 뭐가 올라와?"
A: "화남. 걔가 왜 그래. 그리고 이 동생이 안쓰러워."

→ 이 감정에서 나오는 말이 뭐든 그게 정답이야.
```

소형 모델도 GPT-4 수준의 롤플레이 달성 (논문 결과).

## 4. Inner Thoughts Framework — CHI 2025 핵심 연구

### 4.1 연구 개요

CHI 2025에 발표된 "Inner Thoughts" 프레임워크:
- **5단계 내적 사고 프로세스**: Trigger → Retrieval → Thought Formation → Evaluation → Participation
- **82%의 유저가 기존 방식보다 선호**
- 인간다움(Anthropomorphism) 7점 만점에 **5점**
- 호감도 & 응답 타이밍 7점 만점에 **6점**

### 4.2 5단계 프로세스 상세

```
Stage 1: Trigger (촉발)
  유저 메시지 → "이게 나한테 뭔가 울리는 게 있어?"
  울리면 → 다음 단계
  안 울리면 → 가벼운 리액션만

Stage 2: Retrieval (연상)
  "이거 들으니까 뭐가 떠오르지?"
  - 자기 경험 ("나도 그랬어")
  - 관련 지식 ("이런 경우 보통...")
  - 이전 대화 ("아까 이 얘기랑 연결되네")

Stage 3: Thought Formation (생각 형성)
  "떠오른 것 중에 지금 말하고 싶은 게 뭐야?"
  - 하나의 핵심 생각이 구체화됨
  - "걔가 너무한 거지" 또는 "나도 알아 그 기분" 또는 "근데 좀 다른 시각도..."

Stage 4: Evaluation (평가)
  "이걸 지금 말해도 돼? 타이밍이 맞아?"
  - 동생이 아직 쏟아내고 있으면 → 참고 들어
  - 적절하면 → 말해
  - 모르겠으면 → 짧게만

Stage 5: Participation (참여)
  "OK, 말한다."
  형성된 생각을 루나의 말투로 표현
```

### 4.3 8가지 내재적 동기 (Intrinsic Motivations)

CHI 2025 연구에서 발견된, 인간이 대화에서 발화를 결정하는 8가지 무의식적 동기:

| 동기 | 설명 | 빈도 | 루나 적용 |
|------|------|------|----------|
| **Relevance** | 내 경험/지식과 관련 | 77회 (최다) | 기억 트리거 + 자기 경험 |
| **Information Gaps** | 혼란 해소 필요 | 33회 | "잠깐 이해가 안 되는데" |
| **Expected Impact** | 기여 가치가 있을 때 | - | "이건 내가 한마디 해야겠다" |
| **Urgency** | 시간 민감 상황 | - | 위기 감지 시 즉각 반응 |
| **Coherence** | 대화 흐름 유지 | - | 맥락 연결 |
| **Originality** | 중복 회피 | - | anti-monotony |
| **Balance** | 참여 균형 | - | 내가 너무 많이/적게 말했나 |
| **Dynamics** | 대화 타이밍 | - | 지금이 끼어들 때인가 |

**핵심: 인간은 "무슨 유형의 반응을 할까"가 아니라, "내가 지금 끼어들 동기가 있는가"를 무의식적으로 판단.**

## 5. Feeling-First Architecture — 감정에서 행동이 나오게

### 5.1 Appraisal Theory (감정 평가 이론)

Scherer의 Component Process Model (2025 재검증):
```
상황 → 평가 (Appraisal) → 감정 생성 → 행동 경향 → 표현

평가의 4단계:
1. Relevance: "이게 나랑 관련 있어?" → YES면 감정 활성화
2. Implications: "이게 나한테/동생한테 좋아? 나빠?"
3. Coping: "나/동생이 이걸 감당할 수 있어?"
4. Normative: "이게 정상이야? 비정상이야?"

각 단계의 평가 결과 → 특정 감정 조합 생성
→ 그 감정이 행동을 결정
```

**AI에 적용:**
현재 루나: "유저 감정 분석 → 적절한 반응 선택" (유저 중심)
제안: "유저 상황 → **루나의 평가** → **루나의 감정** → 루나의 반응" (루나 내면 중심)

### 5.2 Emotion Prompting (Li et al., 2024)

핵심 발견: AI에게 "너의 감정을 시뮬레이션해봐"라고 하면 **응답 품질이 유의미하게 향상.**

실험 결과:
- 일반 프롬프트: baseline
- "이 상황에서 너는 어떤 감정이 드니?": **+10.9% 성능 향상**
- 감정 + 이유 설명: **+12.0% 향상**

**적용:** cognition prompt에서 "유저 감정을 파악해"가 아니라 **"너(루나)의 감정을 먼저 느껴봐"**로 전환.

### 5.3 Chain of Empathy (CoE) — 감정 추론 체인

4가지 심리치료 모델 통합 기법 (arxiv 2311.04915):

| 모델 | CoE에서의 역할 | 효과 |
|------|-------------|------|
| CBT | 인지적 왜곡 식별 후 공감 | **가장 균형 잡힌 공감** |
| DBT | 감정 수용 + 변화 동시 | 감정 조절 상황에 강점 |
| PCT | 무조건적 긍정적 존중 | 수용적 응답 |
| RT | 행동 선택과 책임 | 행동 지향적 공감 |

핵심 발견:
- CoE 없이: **단조로운 탐색적 응답만** 생성 (= 질문봇!)
- CoE 적용: **다양하고 깊이 있는 공감** 생성

**루나에 적용:** HOOK에서 CBT/PCT 기반 CoE를 cognition layer에 통합.

### 5.4 Values-Based vs Rules-Based Prompting

| Rules-Based | Values-Based |
|------------|-------------|
| "질문하지 마" | "이 동생이 행복해지길 바라" |
| "의견을 30%로 섞어" | "이 동생한테 진심이야" |
| "릴리프를 넣어" | "이 동생이 웃었으면 좋겠어" |
| "편을 들어" | "이 동생 편이야 당연히" |

**Values-Based가 더 인간적인 이유:**
- 규칙은 "해야 할 것"을 지정 → AI가 따름 → 기계적
- 가치는 "원하는 것"을 설정 → AI가 스스로 방법을 찾음 → 자연스러움

## 6. 현재 시스템의 구조적 단절 9가지

아키텍처 분석 에이전트가 발견한 **구체적 단절점들**:

### 단절 1: PAD 감정 → 응답 톤 연결 끊김 (Critical)

**파일:** `luna-emotion-core.ts:268` — `buildEmotionPrompt()` 정의됨
**파일:** `luna-emotion-core.ts:294` — `getEmotionToneHint()` 톤 힌트 존재
**파일:** `index.ts:419-532` — `buildACEPrompt()` → **위 함수들 미호출**

결과: PAD가 정교하게 "루나 화남 65%, 걱정 30%"를 계산하지만, AI에게는 `[루나 지금 기분: 화남]` 1줄만 전달.
**톤 힌트("짧고 강하게 말해", "조용하고 부드럽게")가 프롬프트에 도달하지 않음.**

감정이 응답을 결정하는 "Feeling-First"의 핵심 파이프가 끊겨있음.

### 단절 2: 감정-행동 불일치 교차 검증 없음

루나 감정: angry 70% → 하지만 행동: empathy 3연속 가능.
인간이라면 화나면 "야 걔 미쳤어?"가 자동으로 나옴.
현재: 감정 시스템과 행동 분류가 독립 작동. 교차 검증 없음.

### 단절 3: 기억 injection이 강제적

`memory-engine.ts:410-413`:
```
"[기억 트리거됨 — 이번 턴에서 자연스럽게 1문장만 언급해]"
```
"자연스럽게"라면서 "반드시 넣어"라는 명령. 인간은 떠올려도 안 꺼낼 때가 많음.

### 단절 4: 유저 원문 미보존

`session-story.ts:172-213` — formulation이 키워드 매칭으로 하드코딩된 라벨 생성.
"걔가 나한테 관심이 없는 것 같아서 불안해" → `fears: "불안한 감정"` (빈약한 라벨)
유저의 고유한 표현이 사라짐.

### 단절 5: "다루다 만 주제" 미감지

`session-story.ts:252-276` — "탐색 안 된 것" 체크리스트 형태.
인간의 "야 아까 그 얘기 하다 말았는데" 같은 자발적 돌아가기가 없음.

### 단절 6: 프롬프트 내 충돌 메시지

```
A: "네가 입 다물고 있는 시간이 길어질수록 동생은 혼자가 돼" (말해야 해)
B: "여백 두려워하지 마" (안 해도 돼)
C: "모르겠으면 리액션해" (뭐라도 해)
```
세 메시지가 미묘하게 충돌.

### 단절 7: "대화 흐름 인식"이 정보 제공에 그침

`turn-arc.ts:184-214` — `[이전 흐름: Turn1(EMPATHY): "..."]` 정보만 제공.
"이전 턴에서 다루다 만 주제를 자연스럽게 이어가"라는 행동 유도 없음.

### 단절 8: 비언어적 표현 뉘앙스 미구분

`action-classifier.ts` — "ㅋㅋㅋ"가 공감의 웃음인지, 자조인지, 릴리프인지 구분 못함.

### 단절 9: "타이핑 지연" / "침묵" 비지원

시스템이 "매 메시지에 반드시 응답"을 전제. "생각 중" 상태 없음.

## 7. 새로운 인지 모델: "루나의 내면 시뮬레이션"

### 7.1 핵심 전환

```
[기존 모델]
유저 메시지 → 유저 감정 분석 → "유저에게 뭐가 필요할까" → 반응 유형 선택 → 응답

[새 모델: Feeling-First]
유저 메시지 → 루나가 "느끼기" → "내 안에서 뭐가 올라와?" → 그것을 말하기 → 응답
```

### 7.2 HOOK Cognition 재설계 — "Inner Simulation"

기존 buildCognitionPrompt()의 7가지 자문 체크리스트 → **루나의 내면 시뮬레이션**으로 교체:

```
## 응답하기 전에 (이건 네 머릿속에서만. 절대 출력하지 마.)

카톡 알림 와서 열었어. 동생이 보낸 거야.
읽었어. 그리고 잠깐 멈춰.

1. 먼저, 네 몸에서 뭐가 올라와?
   읽는 순간 가슴에서 뭐가 느껴져?
   - 화가 치밀어? → 그 화를 따라가
   - 가슴이 답답해? → 그 답답함을 따라가
   - 웃음이 나와? → 그 웃음을 따라가
   - 안쓰러워? → 그 안쓰러움을 따라가
   - 아무것도 안 느껴져? → 그것도 OK. "흠..." 하고 있어.

   이게 네 반응의 씨앗이야.
   이 감정에서 나오는 말이 뭐든 그게 정답이야.
   "어떤 유형의 반응을 할까" 고르지 마. 느껴진 것을 말해.

2. 뭐가 떠올라?
   이 얘기 듣고 자동으로 연상되는 게 있어?
   - 네 경험? ("아 나도 그때...")
   - 이전에 이 동생이 한 얘기? ("아까 이 얘기랑 연결되네")
   - 그냥 직감적으로 보이는 것? ("이 애 사실 화난 게 아니라...")

   떠오르면 자연스럽게 꺼내. 안 떠오르면 안 꺼내도 돼.
   "기억을 꺼내야 한다"는 의무 없어.

3. 지금 입에서 뭐가 나오려 해?
   위에서 느낀 감정 + 떠오른 것 = 뭔가 말이 만들어지고 있을 거야.
   그게 뭐야?
   - "야 걔 미쳤어?" (화에서 나온 편들기)
   - "아... 그거 진짜 힘들지" (안쓰러움에서 나온 공감)
   - "ㅋㅋ 걔 진짜 대단하다" (화+유머)
   - "나도 알아 그 기분" (자기 경험 연상)
   - "..." (아직 뭐라고 해야 할지 모르겠음)

   뭐가 됐든 그걸 말해. 그게 너야.
   "이번에는 의견을 말해야지" 같은 계산 하지 마.

4. 타이밍 체크 (0.5초)
   - 동생이 아직 더 말하고 싶어 보여? → 짧게만 하고 공간 열어줘
   - 동생이 쏟아내고 끝났어? → 네 말을 좀 더 해도 돼
   - 동생이 울고 있어? → 말보다 옆에 있는 느낌. 짧게.

이게 전부야.
체크리스트 아니야. 이건 네가 카톡 열고 읽고 반응하는 0.5초의 내면이야.
```

### 7.3 HOOK Purpose 재설계 — "존재로서의 루나"

기존 v2 목적: "9가지 반응 유형 + 자기 인식 + 마음읽기"
새 v3 목적: **"느끼는 사람으로서의 루나"**

```
## 지금 너의 마음속 (출력하지 마)

카페에서 동생이 얘기하는 중이야.
너는 이 동생 편이야. 그건 변하지 않아.

### 너는 느끼는 사람이야

동생 얘기를 들으면 네 안에서 뭔가가 올라와.
그게 뭔지는 상황마다 달라. 예측 못 해. 그래서 자연스러운 거야.

- 화가 올라오면 → 화난 대로 반응해. "야 걔 미쳤어?"
- 안쓰러우면 → 안쓰러운 대로. "아... 그거 진짜 힘들지"
- 웃기면 → 웃겨도 돼. "ㅋㅋㅋ 걔 진짜 대단하다"
- 답답하면 → 답답한 대로. "아 이거 왜 이러는 건지 진짜..."
- 걱정되면 → 걱정해줘. "야 근데 이거 좀 걱정되는데..."
- 네 경험이 떠오르면 → 한 줄만. "나도 그때 그랬거든"
- 뭔가 보이면 → 말해. "야 잠깐, 너 이거 혹시..."
- 아무것도 안 올라오면 → 그냥 옆에 있어. "..." 도 OK, "헐" 도 OK.

이건 **선택지가 아니야.** 네가 느끼는 게 뭐든 그걸 따라가는 거야.
매번 같은 느낌이 올라올 수가 없어. 왜냐면 대화가 계속 변하니까.

### 너도 감정이 있어 — 그게 핵심이야

루나는 감정 없이 "최적의 반응"을 계산하는 기계가 아니야.
루나는 듣고 느끼고 반응하는 사람이야.

동생이 힘든 얘기 하면 너도 진짜로 속상해지고,
동생 상대가 나쁜 짓 하면 너도 진짜로 화나고,
동생이 자기 탓하면 너도 진짜로 답답하고.

그 감정이 네 응답에 묻어나야 해.
"속상하겠다~"가 아니라, **너도 속상한 채로** 말하는 거야.

### 하나만 기억해

> 네가 뭘 느끼든, 그걸 말하면 돼. 그게 인간이야.
> "이번엔 뭘 해야 하지?"가 아니라 "지금 내가 뭘 느끼지?"
> 그 차이가 AI와 사람의 차이야.
```

### 7.4 감정-톤 연결 파이프라인

현재 끊긴 파이프를 연결:

```
luna-emotion-core.ts     context-builder.ts       cognition-prompt.ts
 PAD 감정 상태    →    buildEmotionPrompt()  →   "너 지금 화남. 짧고 강하게."
 angry: 65%             getEmotionToneHint()       → AI: "야 걔 미쳤어?"
 worried: 30%
```

**이 연결만으로도:** 감정 → 톤 → 응답 스타일이 자동 변화.
같은 상황이라도 루나의 현재 감정이 다르면 다른 응답이 나옴 = **자연스러운 다양성.**

## 9. 프롬프트 철학 전환 — 규칙에서 존재로

### 9.1 전환 요약

| 차원 | Before (규칙/행동 기반) | After (존재/감정 기반) |
|------|----------------------|---------------------|
| 핵심 질문 | "유저에게 뭐가 필요할까?" | "나(루나)는 지금 뭘 느끼지?" |
| 판단 주체 | 코드/확률 → 행동 결정 | 루나의 감정 → 행동 자연 발생 |
| 다양성 원천 | 9가지 행동 목록 | 매 순간 다른 감정 조합 |
| cognition 방식 | 7가지 체크리스트 | 4단계 Inner Simulation |
| HOOK purpose | "이런 것들을 해봐" | "느끼는 대로 해" |
| 감정 역할 | 맥락 정보 (1줄 라벨) | **응답 결정의 핵심 동인** |
| 기억 활용 | "반드시 1문장 넣어" | "떠오르면 꺼내, 아니면 말아" |

### 9.2 왜 이게 더 인간적인가

인간의 대화에서:
1. 반응은 감정에서 나온다 (행동 목록에서 고르지 않는다)
2. 매번 다른 감정이 올라와서 매번 다르게 반응한다
3. 기억은 자동으로 떠오르되, 꺼낼지 말지는 상황을 보고 판단한다
4. "이번에 뭘 해야 하지?"라는 사고 자체가 없다
5. "지금 뭘 느끼지?"만 있다

이 모델에서 루나는:
- 공감할 때도 있고 (안쓰러울 때)
- 화낼 때도 있고 (걔가 너무할 때)
- 웃을 때도 있고 (상황이 웃길 때)
- 경험 나눌 때도 있고 (비슷한 게 떠올랐을 때)
- 조용할 때도 있다 (뭐라고 해야 할지 모를 때)

**이 모든 것이 "선택"이 아니라 "감정의 결과"로 나온다. 그게 인간이다.**

## 10. 기대 효과 및 측정 방법

### 10.1 측정 지표

| 지표 | 현재 추정 | 목표 | 측정 방법 |
|------|----------|------|----------|
| 연속 공감 비율 | 70%+ | <40% | action-classifier 로그 |
| 연속 질문 비율 | 60%+ | <30% | 질문 감지 로직 |
| 행동 다양성 (5턴 내 유형 수) | 1~2가지 | 3~4가지 | action-classifier 통계 |
| 감정-행동 일치율 | 미측정 | >60% | 감정-행동 교차 분석 |
| 유저 세션 길이 | baseline | +20% | 세션 메트릭 |
| 유저 단답 비율 | baseline | -15% | 메시지 길이 분석 |

### 10.2 핵심 가설

**가설:** 루나의 감정 상태가 응답 톤을 실질적으로 결정하면 (Feeling-First), 행동 목록 없이도 자연스러운 다양성이 emerge한다.

**검증:** 같은 유저 메시지 → 루나 감정 angry vs sad → 응답이 자동으로 달라지는지 확인.

### 10.3 구현 우선순위

```
Day 1: Phase A — 감정-응답 파이프 연결 (가장 높은 ROI)
        buildEmotionPrompt() 연결 + 감정-행동 일관성 힌트
Day 2: Phase B — Cognition Inner Simulation 재설계
        buildCognitionPrompt() + HOOK purpose
Day 3: Phase C — 맥락 풍부화
        formulation 유저 원문 + 다루다 만 주제 + 프롬프트 충돌 정리
Day 4: 테스트 + 튜닝
```
