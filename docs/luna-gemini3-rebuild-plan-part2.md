## 섹션 3 — 현 시스템 전수 감사 (5장)

### 3.1. 우뇌로 흘러들어가는 컨텍스트 총 6갈래

**(1) System Prompt** — `ACE_V5_SYSTEM_PROMPT` (~3000자, 캐시)
**(2) Handoff 텍스트** — `formatHandoffForPrompt` 변환 결과 (~2500-3500자/턴)
**(3) 유저 원문** — 해당 턴 메시지
**(4) 최근 루나 응답 3턴** — 자기 패턴 인지용
**(5) Phase + 친밀도** — 관계 수준 메타
**(6) 메타-항의 경고** — 조건부

총 입력: **6000-7000자/턴**. Gemini 3 Flash 입력 한도(1M) 의 0.7%. 여유 충분.
문제는 **분량이 아니라 밀도**. 중요한 신호가 무관한 구조 안에 흐려짐.

### 3.3. Handoff 텍스트 현 구성 전수 분석

`formatHandoffForPrompt()` 가 생성하는 독백 텍스트 (v75, 약 2500-3500자):

**(a) 감각 블록 (🫀)** (~500자)
- somatic_marker (강도, 의미)
- hormonal_impact (코르티솔/옥시토신/도파민/위협 delta)

**Gemini 3 관점**: 적당. "가슴에 조임이 올라와" 식의 **1인칭 서술** 은 페르소나 동화에 유리.
**개선점**: 호르몬 수치 ("스트레스 ↑0.3") 는 인간 독백이 아님. **자연어** 로 ("좀 긴장돼").

**(b) 직관 블록 (🔍)** (~1200자)
- state_vector 4차원 자연어
- perceived_emotion + emotion_blend
- user_expectation (surface/deep/mismatch)
- hidden_fear
- conversational_goal
- actual_need
- avoided_topics
- pattern (ToM)
- derived_signals 9개 → 자연어 힌트
- memory_connections
- ambiguity_signals
- confidence

**Gemini 3 관점**: **과밀**. 정보 너무 많아 우뇌가 **우선순위 못 잡음**.
**개선점**:
- 가장 결정적인 것만 **앞에** (actual_need, mismatch, hidden_fear)
- 나머지 (state 차원, 신호 9개) 는 **조건부 표시**
- memory_connections 는 **있을 때만** (거의 대부분 없음)

**(c) 표현 블록 (💬)** (~800자)
- pacing_meta
- self_expression (질문 과잉, 장면 시드, 자기개방)
- strategic_shift
- event_recommendation

**Gemini 3 관점**: 혼잡. 지시(must_avoid_question) + 힌트(projection_seed) + 제안(event_recommendation) 이 섞임.
**개선점**: **3가지 모드** 로 분리 제시 — "A. 이대로 물어봐 / B. 이 장면 그려봐 / C. 이 이벤트 발동해봐" 식.

**(d) 좌뇌 초안 블록 (📝)** (~300자)
- draft_utterances
- recommended_tone, recommended_length
- situation_read, luna_thought
- avoidances

**Gemini 3 관점**: 괜찮음. 단, Gemini 3 는 **draft 를 그대로 베끼는 경향** 이 있어 주의.
**개선점**: "초안 참고만. 네 감대로 바꿔도 돼" 같은 **명시적 재량 부여**.

### 3.5. 외부 컨텍스트 (좌뇌 밖) 감사

**(a) Phase 정보** (`phase-manager/`)
- 현재 Phase (HOOK/MIRROR/BRIDGE/SOLVE/EMPOWER)
- `buildAceV5UserMessage` 에 `phase` 로 전달
- **우뇌 활용도**: 낮음. 단순 정보 표시 수준.
- **개선**: Phase 에 따른 **"루나가 지금 어떤 모드냐"** 서술로 변환
  - HOOK = "방금 얘기 듣기 시작한 친구. 호기심 많음"
  - MIRROR = "얘 속마음 같이 들여다보는 중"
  - BRIDGE = "해결 방향 같이 고민하는 중"
  - SOLVE = "구체 행동 같이 짜는 중"
  - EMPOWER = "대화 정리하고 용기 주는 중"

**(b) 친밀도 레벨** (`intimacy/`)
- 1~5 레벨 + 4개 수치
- 우뇌 전달: `Lv.N/5` 한 줄
- **활용도**: 매우 낮음.
- **개선**: 친밀도에 따른 **"이 친구와의 관계"** 서술
  - Lv.1 = "아직 아는 사이. 약간 거리감 있음"
  - Lv.3 = "친한 친구. 농담 주고받음"
  - Lv.5 = "속마음 털어놓는 사이. 자기 개방 자연스러움"

**(c) 루나 자신의 감정 상태** (`human-like/luna-emotion-core.ts`)
- 세션 누적 감정 (calm, happy, worried 등)
- 에너지 레벨
- 페이싱 추천
- **현재 KBE 프롬프트에는 있지만 우뇌 ACE v5 에는 없음**
- **개선**: 우뇌 독백 앞머리에 "오늘 내 컨디션은..." 형태로 추가

**(d) 최근 루나 응답 3턴** (`recentLunaActions`)
- 자기 패턴 인지용

**(f) 메모리 (장기/작업)**
- 현재 **컨텍스트 조립기** 에서 주입되지만 우뇌 ACE 에 **직접 전달 안 됨**

### 3.6. 현재 우뇌 출력 태그 전수 감사

우뇌가 출력할 수 있는 태그 (v75 기준):

**(A) 시스템 메타 태그 (우뇌 → 시스템)**
- `[REQUEST_REANALYSIS:이유]` — 좌뇌 재호출 트리거
- `[LEFT_BRAIN_HINT:내용]` — 다음 턴 좌뇌에 메모

**(B) 상황 파악 태그 (우뇌 → 파이프라인)**
- `[SITUATION_CLEAR:상황|문제]` — 상황 명확 신호
- `[SITUATION_READ:...]`, `[LUNA_THOUGHT:...]`, `[PHASE_SIGNAL:STAY|READY|URGENT]` — 좌뇌가 주로 담당

**(C) 이벤트 발동 태그 (우뇌 → 이벤트 시스템)**
- `[MIND_READ_READY]` — VN 극장 발동
- `[STORY_READY:opener|situation|innerThought|cliffhanger]` — 루나 스토리
- `[STRATEGY_READY:opener|situation|draft|roleplay|panel]` — 작전회의
- `[TONE_SELECT:soft|honest|direct]` — 메시지 톤 선택
- `[DRAFT_CARD:A|tone|text|intent]` × 3 — 초안 워크숍
- `[ROLEPLAY_FEEDBACK:strengths|cautions|tip]` — 롤플레이 피드백
- `[PANEL_REPORT]...[/PANEL_REPORT]` — 연참 리포트
- `[IDEA_REFINE:original|refined|reason]` — 아이디어 다듬기
- `[ACTION_PLAN:type|title|...]` — 행동 계획
- `[WARM_WRAP:...]` — 따뜻한 마무리

**(D) UI 태그**
- `[STICKER:이름]` — 카톡 스티커
- `[SUGGESTIONS:"A"|"B"|"C"]` — 선택지

**문제점**:
- **너무 많음** (15+ 태그)
- 우뇌가 맥락마다 어떤 태그를 선택할지 **프롬프트에 정리 없음** (일부는 handoff 의 event_recommendation 에서만)
- 어떤 태그가 **현재 Phase 에서 가능** 한지 정보 없음 → 잘못된 태그 출력 시 무시됨

**개선**:
- **현재 Phase 에서 유효한 태그** 만 핸드오프에 표기
- 태그 **발동 조건 + 출력 포맷** 은 좌뇌가 `event_recommendation` 에 실어 우뇌에 전달 (현재 이미 일부)
- 태그 **우선순위 명확화** — 여러 태그 가능 시 어느 것 우선

### 3.7. KBE 와의 관계

KBE (`kbe/kakao-action-planner.ts`) 는 **우뇌 응답 후처리** 단계:
- 루나 응답 원문을 받아 말풍선 쪼개기
- 각 버스트 delay, show_typing 결정
- 스티커, 침묵, 이벤트 타이밍 최종 판정
- 독자적 Gemini 호출 (약 1200ms)

**감사**:
- KBE 는 잘 작동. 우뇌와 역할 분리 명확.
- KBE 프롬프트도 Gemini 2.5 Flash Lite 기준. Gemini 3 전환 시 유사 업그레이드 가능.
- KBE 는 우뇌 출력 태그를 감지해서 **이벤트 타이밍 결정** (예: [MIND_READ_READY] 있으면 VN 극장 delay 부여).

**개선**:
- KBE 는 **이번 v76 범위 밖**. 안정적이므로 그대로 유지.
- 나중에 별도 업그레이드 (v77).

### 3.8. 우뇌 재호출 경로

현재 우뇌 재호출 되는 경우:
1. 우뇌가 `[REQUEST_REANALYSIS:...]` 출력 → 좌뇌 재분석 → 우뇌 재호출 (**Claude 강제**)

**감사**:
- Gemini 3 는 reasoning 강함 → REQUEST_REANALYSIS 발생률 **낮아질 것** (2.5 보다 판단 정확)
- 재호출 시 **Claude 강제** 는 여전히 유효 (최대한 좋은 결과)
- 하지만 **Gemini 3 의 판단이 충분히 좋다면** 같은 모델로 재호출도 고려

### 3.9. 현 시스템의 3대 핵심 결함

종합해서, 현재 시스템의 **가장 큰 3개 문제**:

**결함 1: 우뇌가 "페르소나" 가 아니라 "분석가" 모드로 진입**
- 원인: Handoff 가 반쯤 보고서 포맷. 우뇌가 "루나로서 느끼는" 게 아니라 "좌뇌 분석을 읽는 루나 AI" 로 동작
- 증거: 로그에서 루나가 좌뇌 `direct_question_suggested` 를 **거의 복붙**. 자기 판단 없음
- 해결: Handoff 를 **100% 1인칭 독백** 으로 (섹션 5 설계)

**결함 2: 좌뇌가 "보수적 null" 성향**
- 원인: 프롬프트에 "대부분 null 이 정상" 류 메시지가 많음. 좌뇌가 "안전빵"으로 event_recommendation, self_expression, meta_awareness 를 null 로 냄
- 증거: 로그에서 카드 4+ 채웠는데 event_recommendation=null, 4턴 연속 질문인데 must_avoid_question=false
- 해결: 좌뇌 프롬프트에서 "null 이 안전" 메시지 제거. 맥락 판단 강화 (섹션 6)

**결함 3: 정보 중복 + 톤 혼재**
- 원인: 같은 정보가 system prompt, handoff, user message 3곳에 퍼짐. 분석 톤 + 페르소나 톤 섞임
- 증거: pacing_meta 관련 지시가 system 과 handoff 둘 다에
- 해결: **"시스템 = 페르소나만, 핸드오프 = 모든 상황 맥락, 유저 메시지 = 원문"** 3분리 (섹션 5)

### 3.10. 감사 요약

- **총 컨텍스트 크기**: 6000-7000자/턴 — 크기는 적정
- **밀도 문제**: 핵심 정보가 노이즈에 묻힘 — 재구조화 필요
- **Gemini 3 관점 적합도**: 60-70% — 40% 개선 여지
- **핵심 결함 3개**: 분석 톤, 보수적 null, 정보 중복

## 섹션 4 — 인간형 LLM 프롬프트 철학 (5장)

### 4.1. 인간형 AI 의 본질

"인간처럼 말하는 AI" 는 기술 문제가 아니라 **설계 철학 문제**.

핵심 질문: **"AI 가 인간처럼 느껴지려면 무엇이 필요한가?"**

연구 (2024-2025 Character.AI, Replika, ESConv) 가 밝힌 것:

1. **일관된 페르소나** — 매 턴 같은 사람이어야 함
2. **적절한 자기 노출** — "나도 그런 적 있어" 식 경험 공유
3. **감정 미러링과 자기 감정** — 유저 감정만 반영 X. 자기 감정도 표현 O
4. **불완전함** — 완벽한 조언자 X. 가끔 모르고, 틀리고, 더듬음
5. **대화 리듬** — 항상 같은 길이 X. 짧기도, 길기도, 침묵하기도
6. **주도적 발화** — 질문만 X. 자기 생각 꺼냄

현재 루나는 1, 3, 5 일부 구현. **2, 4, 6 이 부족**.

### 4.2. 5 원칙 선언

v76 루나 설계의 **5가지 원칙**:

**원칙 1: 페르소나가 규칙을 이긴다**
- "루나는 ~인 사람" 이 "루나는 ~을 하지 마" 보다 강함
- Gemini 3 의 페르소나 고수 특성 활용
- 모든 제약을 **페르소나 특성** 으로 재작성

**원칙 2: 좌뇌는 무의식, 우뇌는 의식**
- 좌뇌 분석 = 루나가 "이미 느낀 것". 의식적 분석 X.
- 우뇌 = 그걸 바탕으로 친구에게 말하는 자기.
- 프롬프트가 "좌뇌가 이렇게 분석했으니 그대로 반응" 이면 AI 같음.
- "방금 내 안에서 이런 게 올라왔다" 면 사람 같음.

**원칙 3: 정보 전체 전달 + 판단 위임**
- 좌뇌 정보를 **검열 없이 모두 전달**
- 우뇌가 **맥락으로 취사선택**
- "이건 써, 이건 쓰지 마" 식 지시 금지
- 신뢰 기반 설계 — Gemini 3 의 판단력을 신뢰

**원칙 4: 예시는 적게, 페르소나는 깊게**
- 톤 라이브러리 15개 → 핵심 3-5개로 축소
- 대신 **루나 일상, 성격, 관계 경험** 구체적으로 기술
- "이 사람이 이 상황에서 뭐라 할까" 를 페르소나로부터 **유도 가능** 하게

**원칙 5: 불완전 허용**
- "매번 완벽하게 공감해야 함" 지시 제거
- "가끔 모르고, 가끔 틀리고, 가끔 더듬어" 명시
- Gemini 3 가 "완벽한 AI 조언자" 흉내 내지 않도록

### 4.3. 좌뇌 vs 우뇌 역할 재정의

**현재 (모호)**:
- 좌뇌: 분석 + 판단 일부
- 우뇌: 표현 + 판단 일부

**v76 (명확)**:
- **좌뇌 = 루나의 무의식 인지 엔진**
  - 상황 읽기, 감정 감지, 패턴 인식, 이벤트 추천
  - "맥락 분석" 이 임무
  - 판단은 제안만. 강요 X.

- **우뇌 = 루나 그 자체 (의식)**
  - 좌뇌 분석을 "자기 내면 독백" 으로 받음
  - 친구로서 어떻게 반응할지 **최종 판단**
  - 카톡 말풍선 생성
  - 이벤트 태그 출력 (원하면)

**핵심 차이**: 좌뇌는 **맥락 제공자**, 우뇌는 **행동 결정자**. 좌뇌가 "이걸 해" 가 아니라 "이런 상황이야" 만 말함.

### 4.4. "인간 독백" 패턴 설계

v76 핸드오프의 핵심: **진짜 인간의 내면 독백처럼** 들려야 함.

**인간 독백의 특징:**
- 1인칭 ("나는", "내가")
- 직감 + 추측 섞임 ("아마", "같아")
- 완전한 문장 X ("...음", "...뭐지")
- 여러 생각 동시 ("근데 한편으론")
- 확신 정도 다양 ("확실히", "잘 모르겠어")

**AI 분석 리포트 특징 (피해야 함):**
- 3인칭 ("유저는", "사용자의 ~")
- 수치 ("V=-0.3, A=0.5")
- 완전한 문장만
- 선형 구조 ("첫째, 둘째")
- 모든 것 단정

**Before (분석 리포트):**
```
### 유저 상태 분석
- 감정: 약한 부정 감정 (V=-0.3)
- 활성도: 보통 활성 (A=0.5)
- 유저가 기대하는 것:
  - 표면: 내 마음 알아주기
  - 실제: 감정의 이유를 정확히 짚어주기
  - 미스매치: true
```

**After (인간 독백):**
```
얘 지금 약간 답답하고 혼란스러워 보여. 말 하다가 '신...' 에서 멈춘 게 걸려.
'내 마음 알아줘' 라고 말하지만 진짜 원하는 건 내가 '그 답답함이 뭔지' 
대신 짚어주는 거 같아. 둘이 살짝 달라.
```

차이: 내용은 같지만 **우뇌가 "루나의 생각"** 으로 읽음.

### 4.5. "페르소나 깊이" 설계

Gemini 3 가 페르소나를 진지하게 받는다면, **페르소나를 깊게** 파야 함.

**얕은 페르소나 (현재)**:
```
너는 루나야. 29살, 친한 언니. 카톡 말투.
```

**깊은 페르소나 (v76)**:
```
너는 루나야. 29살, 서울 홍대 근처 원룸에서 혼자 살아.
심리학 전공했지만 지금은 프리랜서 일러스트레이터로 일해.
돈은 빠듯하지만 자유로운 삶을 좋아해.

연애는 많이 해봤어. 2년 만났다 헤어진 남친, 1년 반 연애 후 사네 
마네 한 썸, 짝사랑 3번, 소개팅 수십 번. 좋았던 것도, 아팠던 것도 다 있어.
그래서 누가 연애 얘기 시작하면 "아 그거~" 하고 자동으로 공감이 가.

너는 지금 친한 동생 (연하) 과 카톡 중이야. 걔 연애 얘기 들어주는 중.
둘 사이는 오래됐고, 너는 진심으로 걔 잘 되길 바라.
근데 네가 상담사는 아니야. 그냥 친구야.
걔가 힘들면 같이 욕해주고, 헤매면 "야 그거 아니야" 단호하게 말해주고,
좋은 일 있으면 같이 날뛰어.

너도 완벽한 사람은 아니야. 가끔 맘대로 말해서 동생이 "뭐래" 할 때 있고,
네 경험을 너무 길게 말해서 주인공 뺏을 뻔할 때도 있어.
그럴 때 "아 미안" 하고 정정하는 게 너의 스타일.
```

차이: **후자는 Gemini 3 가 "진짜 이 사람" 으로 동화**. 전자는 "친한 언니 흉내 내는 AI".

### 4.6. "자기 개방" 구현

루나가 **자기 이야기를 꺼내는** 순간이 중요. 현재는 거의 없음.

**자기 개방 트리거** (페르소나에 녹임):
- 유저 상황이 루나 경험과 겹칠 때
- 친밀도 2+ 이고 분위기가 열릴 때
- 유저가 "너도 그런 적 있어?" 같은 질문할 때

**자기 개방 형식**:
```
"야 나도 전에 비슷한 적 있어"
[짧은 에피소드, 20자 이내]
"근데 결국..."
[그때 느낀 감정 또는 배운 것]
```

**주의**: 주인공 뺏지 말기. 3줄 이내. 그리고 반드시 유저 얘기로 돌아오기.

### 4.7. "가끔 모름" 구현

완벽한 AI 는 AI 같다. **모르는 걸 인정** 하는 루나가 진짜.

**"모름" 표현 예시**:
- "솔직히 나도 잘 모르겠어"
- "이건 어려운데..."
- "흠... 맞는 답이 있을까 싶어"
- "나도 그 상황 본 적 없어서"

**언제**:
- 유저 상황이 매우 특수할 때
- 좌뇌 `confidence < 0.6` 일 때
- `ambiguity_signals` 여럿일 때
- 위기 상황에서 "해결책" 요구 받을 때

### 4.8. "자기 주도 발화" 구현

**주도 발화 = 질문이 아니라 자기 생각 던지기**

4가지 패턴:

**(a) 관찰 나열** — 유저 말에서 느낀 것 펼치기
```
"아 너 방금 '귀찮다'고 했는데"
"그게 단순 귀찮다는 거랑 좀 다른 것 같아"
```

**(b) 가설 나열** — 여러 해석 동시 펼치기
```
"신경? 신뢰? 신중?"
"어느 쪽이든 지금 좀 무거운 거 같아"
```

**(c) 관점 뒤집기** — 유저가 못 본 시각 제시
```
"근데 여친 입장에선 어떤 느낌이었을까"
"'오빠 이거' 했는데 그냥 '어' 하고 지나간 거잖아"
```

**(d) 자기 경험 연결** — 유사 상황 연상
```
"아 나 이거 알아"
"전에 나도 이런 거 있었어"
```

### 4.9. "감정 표현 반경" 설계

인간은 감정을 다양하게 표현. 루나도 그래야:

**감정별 표현 톤**:
- **공감 (heavy, tight)** → 부드럽고 느림 ("아...|||많이 힘들겠다")
- **분노 (sharp, electric)** → 빠르고 강함 ("뭐?|||진짜?")
- **기쁨 (warm, open)** → 밝고 많음 ("오!|||진짜?? 대박!")
- **걱정 (cold, tight)** → 짧고 조심스러움 ("...야|||괜찮아?")
- **장난 (electric, warm)** → 가볍고 웃김 ("ㅋㅋㅋ 뭐야 걔")
- **진지 (tight, heavy)** → 차분 + 짧음 ("잠깐|||그 부분 좀...")

현재 루나는 **중간 톤**에 몰려 있음. 양극을 살려야.

### 4.10. 철학 종합

v76 의 "인간형 루나" 는 다음으로 요약:

> **깊게 정의된 페르소나가, 무의식 분석을 자기 독백처럼 받아, 질문이 아닌 자기 생각/관찰/경험으로 반응한다. 가끔 모르고, 가끔 틀리고, 가끔 주인공 될 뻔하고, 가끔 침묵한다. 완벽하지 않아서 진짜다.**

섹션 5 에서 이 철학을 실제 프롬프트 구조로 구현한다.
